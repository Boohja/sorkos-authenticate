API Endpoints

Endpoint contract for client projects integrating with Sorkos Login. The current base URL is {{ @request_origin }}.

GET/authorize

Starts the authorization-code flow for a registered client project.

Required Query Parameters

client_id
Public client identifier stored in auth_clients.client_id.
redirect_uri
Callback URL. It must exactly match a row in auth_client_redirect_uris.
response_type
Currently only code.
state
Random value generated by the client project and validated by the client after redirect.

Optional Query Parameters

scope
Requested profile fields. Version 1 expects basic profile/email scopes only.
lang
Preferred UI language. Supported values are en and de, constrained by the client settings.
prompt
Reserved for later consent/account-selection behavior.

Example

{{ @request_origin }}/authorize?client_id=sorkos&redirect_uri=https%3A%2F%2Fsorkos.net%2Fauth%2Fcallback&response_type=code&state=random-csrf-token&lang=en

Responses

  • 200 text/html: valid request and no central login session, showing the provider-selection page.
  • 400 text/html: invalid client, redirect URI, response type, or missing state.

GET/oauth/google/start and /oauth/discord/start

Redirects the user to the selected OAuth provider after validating the pending authorization request.

Responses

  • 302 redirect to the external OAuth provider.
  • Error page if no valid pending authorization request exists.

GET/oauth/google/callback and /oauth/discord/callback

Receives provider callbacks, creates or updates the Sorkos Login user, creates the central auth session, then continues the original authorization request.

Expected Query Parameters

code
Provider authorization code.
state
Provider state generated by Sorkos Login.

Responses

  • 302 redirect back to the original client redirect_uri with code and the original client state.
  • Error page if provider state validation or provider login fails.

POST/token

Exchanges a short-lived authorization code for normalized user information. This request is made server-side by the client project.

Request Fields

grant_type
authorization_code
code
The code returned to the client callback.
client_id
The client identifier.
client_secret
The client secret for confidential server-side clients.
redirect_uri
The same redirect URI used in /authorize.

JSON Response

{
  "user": {
    "id": "usr_abc123",
    "email": "user@example.com",
    "email_verified": true,
    "display_name": "User Name",
    "avatar_url": null,
    "preferred_language": "de"
  }
}

GET/logout

Destroys the central Sorkos Login session. Client projects still handle their own local logout separately.

Responses

  • 200 text/html logout confirmation, or redirect if a return target is added later.
